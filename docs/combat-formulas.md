# 戰鬥公式（社群實測，2026-07-06 網路調查）

模擬器推演「搜尋樹內未來交戰」用的公式。**當下這一步的命中率與預期傷害
一律以遊戲攻擊預測 UI 顯示值為權威**；公式只用於畫面給不出數字的假想
局面（深度 ≥ 2 的節點）。公式屬遊戲機制可寫進程式；代入的數值
（機體/駕駛數值、武裝威力）屬內容，一律來自畫面讀取或 cache。

## 傷害計算式

來源：[wikiwiki.jp/gget ダメージ計算式](https://wikiwiki.jp/gget/%E3%83%80%E3%83%A1%E3%83%BC%E3%82%B8%E8%A8%88%E7%AE%97%E5%BC%8F)
（社群實測，與實測值吻合）。

記號：`PlAtk`＝攻方駕駛攻擊值（射擊/格鬥取武裝對應者）、`PlDef`＝守方
駕駛守備值、`UnAtk`/`UnDef`＝機體攻擊力/防禦力、`POW`＝武裝威力。

```
① 駕駛比率補正   = max(0, (PlAtk − PlDef) / 5000)
② 機體比率補正   = max(0, (UnAtk/10 − UnDef/10) / 5000)
③ 駕駛函數補正   = 1 / (exp(250 × (PlDef − PlAtk) / 100000) + 1)
④ 機體函數補正   = 1 / (exp(25 × (UnDef − UnAtk) / 100000) + 1)
⑤ 基礎傷害      = POW × (① + ② + ③ + ④)

⑥ 攻擊補正 = 100 / (exp((5000 − (UnAtk + PlAtk×2)/10) × 30 / 100000) + 1)
⑦ 防禦補正 = −40 / (exp((5000 − (UnDef + PlDef×2)/10) × 3 / 100000) + 1)
⑧ 戰鬥基本傷害 = ⑤ × (1 + ⑥ + ⑦) / 地形補正

⑨ 傷害增減補正 = 1 + Σ增傷 − Σ減傷      （增減傷合算後才乘）
⑩ 最終傷害     = ⑧ × ⑨ × 防禦行動倍率
⑪ 暴擊時       = ⑧ × ⑨(含暴傷上升) × 防禦行動倍率 × 暴擊倍率
```

- 防禦行動倍率：通常防禦 **0.8**、盾牌防禦 **0.6**（不防禦 1.0）。
- 暴擊倍率：通常 **1.1**、強氣/超強氣 **1.2**、超一擊 **1.3**。

## 命中率計算式

來源：[YouTube 命中率・回避率檢證](https://www.youtube.com/watch?v=TU7XrR38Pqo)
（社群回歸擬合，精度低於傷害式，僅供樹內近似）。

```
命中率(%) = 96.45
          + 0.00732 × 攻方機動力 − 0.00662 × 守方機動力
          + (攻方駕駛對應攻擊值 − 守方駕駛反応值) / 25
          + 雙方能力補正
```

- 反応值主司迴避、機動力雙向影響命中與迴避
  （[ggene.site 反応值解說](https://ggene.site/guide/reaction-value/)）。
- **閃避/防禦行動對命中率的修正值未查到實測數據**——待實機標定：
  同一場交戰比對「選閃避前後」預測 UI 顯示的命中率差。

## 交戰結算順序（2026-07-13 使用者實測案例＋逐案問答定案）

單次交戰（M 攻擊 A；A 方可能有支援防禦 B、支援攻擊者 C×N）依此
順序結算：

1. **攻擊側出手——支援齊射先、主攻後**（案例 13：進攻側支援攻擊
   存在，順序同反擊階段）。有合格支援防禦者時由 B 代替 A 承受——
   且**整輪齊射（含主攻）全部打在 B 身上，第一發擊破 B 也不會讓
   後續打穿到 A**（案例 13）。傷害以 B 的數值計算，預測畫面直接
   顯示 B 承受與擊殺判定。**每次交戰只有一名攔截者**（案例 16，
   誰來攔由遊戲 AI 決定）、整輪只消耗一次攔截次數，且 **miss 不
   消耗**（案例 12，全 miss 則次數保留）。支援防禦有每相位次數、
   相位開始時重置（案例 4）——「先燒攔截次數再殺主目標」的拆網
   打法成立；但**帶支援齊射打有攔截者的目標＝火力灌進同一個承受
   者**，燒攔截應該用單發，齊射留給無攔截的目標。
2. **承受者立即死亡判定**：B 陣亡不取消後續任何結算；**交戰擊殺
   （不論倒下的是 B 還是 A）給主攻者再動**（案例 5；前提是撐過
   反擊階段且再動次數未用完）。
3. **反擊階段，只看主目標（防守方）是否存活**（雙向確認：我方回合
   殺 A 則敵方 C 停手；敵方回合 M 被擊破則我方 N 停手）：
   - **先結算全部支援攻擊**（案例 11；支援機體常帶降防／增傷 debuff，
     先結算主目標的反擊才吃得到效果），**最多 3～4 名同時**（案例 9a，
     上限 3 或 4 待網查）；支援攻擊條件只看「M 在 C 的武裝射程內＋
     C 還有次數」，與 A 能否反擊無關，C 只能選射程夠的武裝（案例 7）。
   - **再結算 A 的應戰反擊**；M 中途被擊破則後續不出手。

**支援資格＝駕駛特性＋機體移動力**（案例 14，確認、非近似）：支援
能力來自駕駛（坐任何機體都帶著走；支援防禦駕駛通常另有「駕駛防禦
機體時防禦能力上升」詞條），支援防禦範圍＝該機體的移動力（移動力
3 就只能防禦 3 格內的我軍）。偵察時要讀駕駛能力欄，不只機體。

**支援攻擊有獨立命中率**（案例 17）：預測畫面可點支援武裝看它自己
的命中%。模擬器目前把支援齊射當必中（solver 簡化），讀取器支援
命中%是 M2 的後續工作。

**齊射擊殺歸屬**（案例 18）：不論致命一擊是主攻還是支援射出的，
交戰擊破都觸發**主攻者**的再動——「這次進攻成功擊破」是判準。
遊戲內破壞數統計可能記給實際擊殺者（依遊戲模式而定，非所有模式
有統計）——對帳鏈用 HUD 破壞數裁決擊殺時留意此差異。

**再動＝整個行動重來**（案例 19）：可再移動、再攻擊、放技能、
施放地圖炮，完整重置。模擬器 acted=False 的建模正確。

**應戰無次數限制**（案例 20）：只要 EN 夠、單位未被擊破，可以在
同一相位連續應戰任意多次。衍生戰術（使用者實證）：夠硬的單位丟進
敵群當「反擊誘餌」，遊戲 AI 會持續攻擊它，靠應戰反殺——搜尋樹
應能自行發現這條線（敵方 policy＋反擊定價都已建模）。

**反擊只打主攻**（案例 21）：A 的應戰只會打 M，支援射手 N 射完
即退場，不會被反擊選中。

**應戰／支援的 EN 經濟**（案例 22）：兩者皆消耗該武裝的 EN，EN
不足該武裝就不可用，但單位有複數武裝、可換低消耗武裝過渡（模擬器
的「射程＋EN 合格才選用」建模正確）。武裝耗能與射程是逐單位內容。
**疑似存在無詞條的每回合 EN 自然回復**（敵我兩相位合計為一回合），
使用者不確定，待網查——若為真，模擬器多回合的 EN 經濟目前偏悲觀
（沒有建模自然回復）。

實測案例：

1. M 攻 A、B 支援防禦，預測顯示 B 承受、M 能擊殺 B、A 反擊：執行後
   B 被擊破，A 的應戰傷害仍打到 M——攔截者陣亡不取消 A 的反擊。
2. 與案例 1 同型（原記錄「預測顯示 M 直接擊殺 A、合格的 B 未攔截」
   有誤，2026-07-13 使用者修正：預測實際顯示 M 的傷害被 B 防禦、
   擊殺 B、A 反擊）。原版衍生的「觸發條件未知」異例不存在。
3. M 攻 A、B 支援防禦＋C 支援攻擊，預測顯示 M 擊破 B 但會被 A＋C
   擊破：執行後 B 陣亡，A 應戰＋C 支援攻擊合力擊破 M。

**防守方選項**（案例 8）：應戰彈窗的所有防守側決策（應戰／防禦／
迴避、支援防禦、支援反擊與其武裝）都是玩家選項，遊戲只是預填預設
值；controller v1 一律核准預設值，選項優化掛 issue #3。solver 把防守
反應當決策節點枚舉是正確建模。我方支援攻擊同樣可設定多人（案例 9a）。

**特殊機制（內容，從能力面板讀取，機制尚未建模）**：

- **進攻盾**（案例 6）：部分支援防禦機體可在「我方進攻時」攔截敵方
  的反擊；一般支援防禦不攔反擊，反擊傷害直接打在攻擊方身上。判別
  與建模掛 issue #22。
- **支援防禦減傷詞條**（案例 10）：部分機體有「支援防禦時減輕 X%
  傷害」能力——減傷是逐機體內容、非全域常數，基礎公式待網查
  （issue #20）。
- **支援攻擊 debuff**（案例 11/15）：降防、特定傷害類型增傷等，
  持續時間通常一回合（我方 phase＋敵方 phase）；敵方 phase 上的
  debuff 會不會延續到我方 phase 待網查或實測。效果本身未建模，
  僅結算順序正確。
- 現階段敵我雙方皆無「同時擁有支援攻擊＋支援防禦」的機體
  （案例 9b；機制上模擬器允許，內容上不存在）。

模擬器殘餘假設：多名支援攻擊者之間的結算順序（現用盤面序；同時
結算故僅影響動畫判讀）、齊射中擊殺的再動記給主攻者、齊射已擊殺
目標後其餘已承諾的射擊不轉移目標（灌進屍體）、攔截者由盤面序
代選（實際是遊戲 AI 決定，當下以預測畫面為權威）。

當下這一步以預測畫面為權威；搜尋樹內由敵方模型假設反應（policy＝
應戰＋一律攔截，與全部實測一致；min＝攔截與否雙態枚舉，保守界）。

## 待實機標定清單

- 閃避行動的迴避率增量（上式的行動補正項）。
- 地形補正表（⑧ 的除數）。
- **支援防禦承受方減傷倍率——存疑（issue #20）**：「由支援者承受」
  已由 2026-07-13 實測案例定案，但現用的 0.5 是 v0 時代的臆測預設，
  無任何實測或社群資料支持，使用者 2026-07-13 明確標記存疑。驗證
  兩路並行：實機（有支援防禦的交戰，預測畫面顯示的對 B 傷害 ÷
  公式基礎傷害）＋網路查證（wikiwiki.jp/gget 等有無實測數據）。
- 支援防禦觸發規則的實證確認（實測皆「有合格支援者即攔截」，但樣本
  少；對帳鏈的 battle_prep 塌陷紀錄持續蒐證）。
- 同時支援攻擊人數上限（3 或 4，網查；`SimParams.max_support_attackers`
  現用 3，issue #21 一併處理）。
- 戰場介面哪裡顯示支援防禦／支援攻擊的剩餘次數（issue #21；拆網
  打法與 bridge 的次數注入都需要）。
- debuff 跨相位持續規則（敵方 phase 上的會不會帶進我方 phase，
  案例 15 使用者不確定；網查或實測）。
- 支援武裝命中%的預測畫面讀取（案例 17：點支援武裝可見；M2
  讀取器後續工作，模擬器目前當必中）。
- **EN 每回合自然回復**（案例 22：疑似無詞條也回、敵我兩相位合計
  一回合回一次；量是固定值還是百分比不明；網查＋實機記兩回合開頭
  同一單位 EN 即可定案）。模擬器未建模，多回合 EN 經濟偏悲觀。
- **地圖炮（MAP 武器）機制**（案例 19 證實存在且再動可施放）：
  範圍規則、是否觸發應戰／可否被支援防禦攔截、多殺與再動的關係
  皆未知；模擬器完全未建模。
- 暴擊率公式（僅知倍率，不知發生率）。
- 命中式常數的本作版本校驗（以攻擊預測 UI 顯示值回歸驗證）。
