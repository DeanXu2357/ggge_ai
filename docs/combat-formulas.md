# 戰鬥公式（社群實測，2026-07-06 網路調查）

模擬器推演「搜尋樹內未來交戰」用的公式。**當下這一步的命中率與預期傷害
一律以遊戲攻擊預測 UI 顯示值為權威**；公式只用於畫面給不出數字的假想
局面（深度 ≥ 2 的節點）。公式屬遊戲機制可寫進程式；代入的數值
（機體/駕駛數值、武裝威力）屬內容，一律來自畫面讀取或 cache。

## 傷害計算式

來源：[wikiwiki.jp/gget ダメージ計算式](https://wikiwiki.jp/gget/%E3%83%80%E3%83%A1%E3%83%BC%E3%82%B8%E8%A8%88%E7%AE%97%E5%BC%8F)
（社群實測，與實測值吻合）。

記號：`PlAtk`＝攻方駕駛攻擊值（射擊/格鬥取武裝對應者）、`PlDef`＝守方
駕駛守備值、`UnAtk`/`UnDef`＝機體攻擊力/防禦力、`POW`＝武裝威力。

```
① 駕駛比率補正   = max(0, (PlAtk − PlDef) / 5000)
② 機體比率補正   = max(0, (UnAtk/10 − UnDef/10) / 5000)
③ 駕駛函數補正   = 1 / (exp(250 × (PlDef − PlAtk) / 100000) + 1)
④ 機體函數補正   = 1 / (exp(25 × (UnDef − UnAtk) / 100000) + 1)
⑤ 基礎傷害      = POW × (① + ② + ③ + ④)

⑥ 攻擊補正 = 100 / (exp((5000 − (UnAtk + PlAtk×2)/10) × 30 / 100000) + 1)
⑦ 防禦補正 = −40 / (exp((5000 − (UnDef + PlDef×2)/10) × 3 / 100000) + 1)
⑧ 戰鬥基本傷害 = ⑤ × (1 + ⑥ + ⑦) / 地形補正

⑨ 傷害增減補正 = 1 + Σ增傷 − Σ減傷      （增減傷合算後才乘）
⑩ 最終傷害     = ⑧ × ⑨ × 防禦行動倍率
⑪ 暴擊時       = ⑧ × ⑨(含暴傷上升) × 防禦行動倍率 × 暴擊倍率
```

- 防禦行動倍率：通常防禦 **0.8**、盾牌防禦 **0.6**（不防禦 1.0）。
- 暴擊倍率：通常 **1.1**、強氣/超強氣 **1.2**、超一擊 **1.3**。

## 命中率計算式

來源：[YouTube 命中率・回避率檢證](https://www.youtube.com/watch?v=TU7XrR38Pqo)
（社群回歸擬合，精度低於傷害式，僅供樹內近似）。

```
命中率(%) = 96.45
          + 0.00732 × 攻方機動力 − 0.00662 × 守方機動力
          + (攻方駕駛對應攻擊值 − 守方駕駛反応值) / 25
          + 雙方能力補正
```

- 反応值主司迴避、機動力雙向影響命中與迴避
  （[ggene.site 反応值解說](https://ggene.site/guide/reaction-value/)）。
- **閃避/防禦行動對命中率的修正值未查到實測數據**——待實機標定：
  同一場交戰比對「選閃避前後」預測 UI 顯示的命中率差。

## 交戰結算順序（2026-07-13 使用者實測案例；案例 2 同日修正）

單次交戰（M 攻擊 A；A 可能有支援防禦 B、支援攻擊 C）依此順序結算：

1. **M 出手**：有合格支援防禦者時由 B 代替 A 承受這一擊——傷害以 B
   的數值計算，B 可被擊破，遊戲預測畫面直接顯示 B 承受與對 B 的擊殺
   判定。目前所有實測案例凡有合格 B 皆攔截，尚無「不攔截」反例。
2. **承受者立即死亡判定**：B 陣亡不取消後續任何結算。
3. **反擊階段，只看主目標（防守方）是否存活**（2026-07-13 使用者
   確認，雙向皆然）：主目標存活 → 主目標的應戰反擊與其支援攻擊者
   都打向攻擊方；主目標被擊破 → 反擊階段整個取消，支援攻擊者自然
   不發動。我方回合（M 攻 A，A 死則敵方 C 停手）與敵方回合（A 攻
   M，M 死則我方 N 停手）對稱成立。

實測案例：

1. M 攻 A、B 支援防禦，預測顯示 B 承受、M 能擊殺 B、A 反擊：執行後
   B 被擊破，A 的應戰傷害仍打到 M——攔截者陣亡不取消 A 的反擊。
2. 與案例 1 同型（原記錄「預測顯示 M 直接擊殺 A、合格的 B 未攔截」
   有誤，2026-07-13 使用者修正：預測實際顯示 M 的傷害被 B 防禦、
   擊殺 B、A 反擊）。原版衍生的「觸發條件未知」異例不存在。
3. M 攻 A、B 支援防禦＋C 支援攻擊，預測顯示 M 擊破 B 但會被 A＋C
   擊破：執行後 B 陣亡，A 應戰＋C 支援攻擊合力擊破 M。

模擬器未經實測的保守假設（`battle/sim.py` docstring 同步記載）：
攔截消耗次數不看命中與否、擊破攔截者同樣給再動、反擊階段主目標先
支援者後（攻擊方中途陣亡即停止後續出手）、支援攻擊者出手只看主目標
存活不看其應戰選項、每次交戰攔截者／支援攻擊者各至多一位。

當下這一步以預測畫面為權威；搜尋樹內由敵方模型假設反應（policy＝
應戰＋一律攔截，與全部實測一致；min＝攔截與否雙態枚舉，保守界）。

## 待實機標定清單

- 閃避行動的迴避率增量（上式的行動補正項）。
- 地形補正表（⑧ 的除數）。
- 支援防禦倍率（「由支援者承受」已由 2026-07-13 實測案例定案；
  承受方的減傷倍率仍未標定，現用 0.5 預設）。
- 支援防禦觸發規則的實證確認（實測皆「有合格支援者即攔截」，但樣本
  少；對帳鏈的 battle_prep 塌陷紀錄持續蒐證）。
- 暴擊率公式（僅知倍率，不知發生率）。
- 命中式常數的本作版本校驗（以攻擊預測 UI 顯示值回歸驗證）。
